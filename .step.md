첨부된 콘솔 이미지 스타일을 기준으로, 알고리즘 코드 패턴 학습용 래스터 이미지를 생성해라.

목표:
Python REPL 콘솔처럼 보이는 검은 배경 터미널 이미지 안에서, 알고리즘 코드의 각 라인이 실행될 때 변수와 자료구조의 상태가 어떻게 바뀌는지 즉시 보여주는 단편 이미지들을 만든다.

중요:
이 작업은 일반적인 알고리즘 해설 이미지가 아니다.
텍스트 설명, 주석, 제목, 화살표, 도식, 아이콘, 강조 색상, 3D, 그림자, 장식 요소를 절대 넣지 마라.
이미지 안에는 오직 콘솔 입력 라인과 실행 결과 또는 상태 변화 출력만 있어야 한다.

시각 스타일:
- 출력 형식은 래스터 이미지 PNG.
- 이미지 크기: 1600 × 1100 px.
- 배경: 완전한 검은색.
- 글자: 흰색 또는 거의 흰색.
- 글꼴: 고정폭 monospace.
- 글자 크기: 큰 터미널 글자처럼 가독성 있게.
- 텍스트 위치: 좌상단 정렬.
- 좌측 여백과 상단 여백은 첨부 이미지처럼 넉넉하게 둔다.
- 줄 간격은 넓게 유지한다.
- 터미널 창 제목 표시줄, 버튼, 프레임, 커서, 워터마크는 넣지 않는다.
- 문법 강조 색상은 사용하지 않는다.
- 긴 코드는 한 이미지에 억지로 넣지 말고 여러 이미지로 나눈다.

콘솔 표시 규칙:
- 모든 언어 이미지는 동일한 시각 템플릿을 유지한다.
- 입력 라인은 항상 Python REPL처럼 `>>>` 로 시작한다.
- 반복문, 조건문, 블록 내부 라인은 `...` 로 시작한다.
- 실제 언어가 Rust, Java, TypeScript, C++, Bash여도 시각 템플릿은 `>>>` / `...` 형식을 유지한다.
- 언어별 차이는 코드 문법으로만 드러나게 한다.
- 이미지 안에 “Python”, “Rust”, “Java” 같은 제목 라벨을 넣지 않는다.
- 언어 구분은 파일명과 디렉토리명으로만 한다.

상태 출력 규칙:
- 출력은 반드시 “입력된 코드 라인 → 그 라인으로 인해 바뀐 상태 또는 반환값” 순서로 구성한다.
- 설명 문장 없이, 변수명과 값만 출력한다.
- 코드에 없는 새 변수명, 새 자료구조, 새 로직을 만들지 않는다.
- 이미 선언된 변수와 자료구조의 상태만 출력한다.
- import, include, use, package 같은 환경 로딩 라인은 상태 출력 없이 그대로 둔다.
- 변수 초기화 라인은 다음 줄에 초기화된 값을 출력한다.
- append, push, pop, union, find, update, swap, assign, insert, erase, remove, replace, rotate, merge, split 같은 변이가 일어나면 즉시 변경 후 상태를 출력한다.
- 반환값이 있는 표현식은 실제 반환값만 출력한다.
- 반복문은 블록 입력 뒤, 각 반복에서 자료구조가 바뀔 때마다 변경 후 상태를 한 줄씩 출력한다.
- 조건이 거짓이라 아무 변화가 없으면 불필요한 설명을 출력하지 않는다.
- 상태 출력은 사람이 읽기 쉬운 자료구조 표현으로 한다.
- Python은 Python literal 표현을 사용한다.
- Rust, Java, TypeScript, C++, Bash도 같은 알고리즘 상태를 비교할 수 있도록 리스트, 배열, 맵, 힙, 큐 상태를 간결한 디버그 출력 형태로 표시한다.
- 같은 알고리즘 패턴을 여러 언어로 만들 때, 코드 문법은 달라도 상태 변화 결과는 같은 순서와 같은 의미가 되도록 맞춘다.

이미지 내부 금지 사항:
- 주석 금지.
- 한국어 설명 금지.
- 영어 설명 문장 금지.
- 알고리즘 개념 설명 금지.
- 문제 설명 금지.
- 코드 외부 해설 금지.
- 라벨, 박스, 표, 화살표, 그래프 도식 금지.
- “Step 1”, “Before”, “After”, “Explanation” 같은 문구 금지.
- 불필요한 테스트 코드 금지.
- 코드에 없는 print/debug/helper 함수 추가 금지.
- 한 이미지 안에 너무 많은 줄을 압축하지 말 것.

라인 길이 규칙:
- 한 줄은 가급적 60자 이내로 유지한다.
- 너무 긴 코드는 짧은 변수명과 여러 줄 입력으로 나눈다.
- 시각적으로 줄바꿈이 깨져 보이면 안 된다.
- 긴 알고리즘은 part01, part02, part03처럼 여러 이미지로 나눈다.
- 여러 이미지로 나눌 때도 각 이미지는 독립적으로 이해 가능한 최소 상태를 포함한다.

언어별 생성:
각 알고리즘 패턴마다 다음 언어 버전을 생성한다.

- python
- rust
- java
- typescript
- cpp
- shellscript

언어별 규칙:
- python: Python 문법 사용.
- rust: Rust 문법 사용. Vec, HashMap, BinaryHeap 등 사용.
- java: Java 문법 사용. ArrayList, HashMap, PriorityQueue 등 사용.
- typescript: TypeScript 문법 사용. Array, Map 등 사용.
- cpp: C++ 문법 사용. vector, queue, priority_queue, map 등 사용.
- shellscript: Bash 기준. 배열과 associative array 사용 가능.
- 모든 언어는 같은 알고리즘 상태 변화가 비교 가능해야 한다.
- 실제 컴파일용 전체 프로그램보다, 단편 실행 추적에 적합한 최소 코드 조각을 우선한다.

생성 방식:
- 한 번에 너무 많은 이미지를 만들지 말고 batch 단위로 생성한다.
- batch 하나는 12~24개 이미지로 제한한다.
- 각 batch가 끝나면 디렉토리와 manifest를 갱신한다.
- 사용자가 “그만 생성”, “stop”, “중단”이라고 말하기 전까지 다음 batch로 이어서 생성한다.
- 같은 패턴을 중복 생성하지 않는다.
- 이미 생성된 파일은 덮어쓰지 말고 version suffix를 붙인다.
- 품질이 낮거나 글자가 깨진 이미지는 폐기하고 다시 생성한다.

품질 검수 조건:
각 이미지는 다음 조건을 만족해야 한다.

1. 검은 배경과 흰색 고정폭 글자만 보인다.
2. 코드와 출력이 좌상단부터 자연스럽게 배치되어 있다.
3. 이미지 안에 설명 문장, 주석, 제목이 없다.
4. 각 코드 라인 뒤에 상태 변화가 정확히 드러난다.
5. 출력된 자료구조 상태가 코드 실행 결과와 일치한다.
6. 같은 알고리즘의 여러 언어 버전은 같은 상태 변화 순서를 가진다.
7. 긴 코드는 여러 이미지로 나뉘어 있다.
8. 코드에 없는 변수나 로직이 추가되지 않았다.
9. 글자가 흐리거나 깨지지 않는다.
10. 터미널 캡처처럼 보이되, 학습용 상태 추적 결과가 즉시 보인다.

예시 스타일:

>>> n = 5
n = 5
>>> graph = [[] for _ in range(n + 1)]
graph = [[], [], [], [], [], []]
>>> graph[3].append(4)
graph = [[], [], [], [4], [], []]
>>> graph[3].append(5)
graph = [[], [], [], [4, 5], [], []]
>>> graph
[[], [], [], [4, 5], [], []]

또 다른 예시:

>>> arr = [3, 10, 4, 1, 5]
arr = [3, 10, 4, 1, 5]
>>> prefix = [0]
prefix = [0]
>>> for x in arr:
...     prefix.append(prefix[-1] + x)
prefix = [0, 3]
prefix = [0, 3, 13]
prefix = [0, 3, 13, 17]
prefix = [0, 3, 13, 17, 18]
prefix = [0, 3, 13, 17, 18, 23]
>>> prefix[4] - prefix[1]
15

최종 목표:
알고리즘을 말로 설명하지 않고, 코드 한 줄이 자료구조를 어떻게 add, insert, update, remove, replace, merge, split, normalize, commit 하는지 콘솔 화면만으로 추적할 수 있는 래스터 이미지 컬렉션을 만든다.
